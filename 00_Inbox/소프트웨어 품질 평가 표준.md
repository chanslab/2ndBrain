#ISO/IEC_25010

ISO/IEC 25010:2023은 소프트웨어 제품의 품질을 평가하기 위한 국제 표준으로, 2011년 버전을 계승하여 **시스템 및 소프트웨어 제품 품질 모델**을 정의합니다. 2023년 개정판은 현대적인 IT 환경(클라우드, 모바일, AI 등)을 반영하여 체계가 더욱 정교해졌습니다.

---

## 1. 주요 변경 사항 및 특징

가장 큰 변화는 기존의 8가지 품질 특성에 **'유연성(Flexibility)'**이 추가되어 총 **9가지 주특성**으로 확장되었다는 점입니다. 또한, '사용성'이 '사용자 경험(User Experience)' 관점으로 보완되었습니다.

---

## 2. 제품 품질 모델 (9대 특성)

소프트웨어 자체의 품질을 측정하는 모델로, 다음과 같은 항목들로 구성됩니다.

| **주특성 (Major Characteristics)**     | **설명**                                |
| ----------------------------------- | ------------------------------------- |
| **기능 적합성 (Functional Suitability)** | 명시된 요구사항을 소프트웨어가 얼마나 정확하게 충족하는가       |
| **수행 효율성 (Performance Efficiency)** | 자원 사용량 대비 처리 속도나 응답 시간의 효율성           |
| **호환성 (Compatibility)**             | 다른 제품과 정보를 교환하거나 동일한 환경에서 공존할 수 있는 능력 |
| **사용성 (Usability)**                 | 사용자가 소프트웨어를 얼마나 쉽고 직관적으로 사용할 수 있는가    |
| **신뢰성 (Reliability)**               | 특정 조건과 시간 내에 오류 없이 기능을 수행할 수 있는 정도    |
| **보안성 (Security)**                  | 데이터와 정보를 보호하고 비인가 접근을 차단하는 능력         |
| **유지보수성 (Maintainability)**         | 제품을 수정, 개선하거나 환경 변화에 적응시키기 쉬운 정도      |
| **이식성 (Portability)**               | 한 환경에서 다른 환경으로 얼마나 쉽게 옮길 수 있는가        |
| **유연성 (Flexibility) `New`**         | 정의된 요구사항 외의 상황이나 미래의 변화에 대응할 수 있는 능력  |

---

## 3. 사용 품질 모델 (Quality in Use)

사용자가 실제 업무 환경에서 소프트웨어를 사용할 때 느끼는 품질을 의미하며, 5가지 특성을 가집니다.

- **유효성 (Effectiveness):** 사용자가 목표를 정확하고 완전하게 달성하는지 여부
    
- **효율성 (Efficiency):** 목표 달성을 위해 소비되는 자원의 양
    
- **만족도 (Satisfaction):** 사용자의 요구와 기대가 충족되어 느끼는 만족감
    
- **위험으로부터의 자유 (Freedom from Risk):** 경제적, 운영적, 환경적 위험을 최소화함
    
- **상황 포괄성 (Context Coverage):** 지정된 상황뿐만 아니라 잠재적 상황에서도 품질을 유지함
    

---

## 4. 도입 시 기대 효과

1. **공통 언어 제공:** 개발자, 테스터, 고객 간의 품질 요구사항에 대한 오해를 줄입니다.
    
2. **체계적 테스트 기준:** 품질 특성별로 테스트 케이스를 설계하여 검수 누락을 방지합니다.
    
3. **객관적 평가 지표:** 소프트웨어 도입 시 BMT(벤치마크 테스트) 등의 객관적 기준으로 활용됩니다.
