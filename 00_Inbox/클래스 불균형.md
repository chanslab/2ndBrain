**클래스 불균형(Class Imbalance)**이란 머신러닝이나 데이터 분석을 할 때, 학습 데이터셋에 포함된 각 클래스(범주)별 데이터의 양이 **현저하게 차이가 나는 상태**를 말합니다.

쉽게 말해, 특정 데이터는 아주 많은데 다른 데이터는 아주 적은 상황입니다.

---

### 1. 주요 개념

데이터셋이 다음과 같이 구성되어 있다고 가정해 보겠습니다.

- **다수 클래스 (Majority Class):** 데이터의 비중이 압도적으로 많은 그룹 (예: 정상 거래 데이터 99%)
    
- **소수 클래스 (Minority Class):** 데이터의 비중이 매우 적은 그룹 (예: 사기 거래 데이터 1%)
    

머신러닝 모델은 기본적으로 "전체 정확도(Accuracy)"를 높이는 방향으로 학습하려는 성질이 있습니다. 데이터가 불균형하면 모델은 단순히 **"모든 데이터를 다수 클래스라고 예측"**해도 99%의 정확도를 얻게 되므로, 정작 중요한 소수 클래스를 제대로 학습하지 못하는 문제가 발생합니다.

---

### 2. 발생하는 주요 사례

우리 주변의 데이터는 의외로 불균형한 경우가 많습니다.

- **금융 사기 탐지:** 대부분은 정상 결제이며, 카드 도난이나 부정 사용은 극히 일부입니다.
    
- **질병 진단:** 특정 희귀병 검사 데이터에서 양성 반응은 음성 반응에 비해 현저히 적습니다.
    
- **제조 공정 불량 탐지:** 공정 시스템이 우수할수록 불량품 데이터는 정상 제품보다 훨씬 적게 발생합니다.
    

---

### 3. 해결 방법

클래스 불균형 문제를 해결하기 위해 주로 다음과 같은 전략을 사용합니다.

|**구분**|**방법**|**설명**|
|---|---|---|
|**데이터 기반**|**언더샘플링 (Undersampling)**|다수 클래스의 데이터를 줄여서 비율을 맞춤|
||**오버샘플링 (Oversampling)**|소수 클래스의 데이터를 복제하거나 생성(SMOTE 등)하여 늘림|
|**알고리즘 기반**|**가중치 부여 (Weighting)**|소수 클래스 데이터를 틀렸을 때 더 큰 벌점을 주도록 설정|
|**평가 지표 변경**|**정밀도, 재현율, F1-score**|정확도(Accuracy) 대신 실제 성능을 잘 보여주는 지표 사용|

---

### 4. 왜 위험한가요?

불균형을 방치하면 모델은 **'게으른 학습'**을 하게 됩니다. 암 환자를 찾아내야 하는 모델이 모든 사람을 '정상'이라고 답해도 수치상의 정확도는 높게 나오지만, 실제 환자를 한 명도 찾아내지 못하는 **치명적인 실패**를 초래할 수 있기 때문입니다.

클래스 불균형을 해결하기 위해 구체적인 알고리즘(예: SMOTE)이나 평가 지표에 대해 더 알고 싶으신가요?
