
### 1. ISO/IEC 25010의 2가지 핵심 모델

이 표준은 소프트웨어 자체의 품질뿐만 아니라, 그것을 사용하는 '경험'까지 평가합니다.

- **제품 품질 모델 (Product Quality Model):** 시스템/소프트웨어 자체의 정적인 특성 (8대 주특성).
    
- **사용 시 품질 모델 (Quality in Use Model):** 실제 사용자가 특정 환경에서 사용했을 때 느끼는 품질 (효과성, 효율성, 만족성, 위험 회피성, 문맥 망라성 등).
    

### 2. 제품 품질 8대 특성 및 주요 부특성 (2011/2023 기준)

기술사 답안지에는 주특성 옆에 최소 2~3개의 부특성을 함께 써주는 것이 고득점 포인트입니다.

| 주특성        | 핵심 의미       | 주요 부특성 (키워드)                              |
| ---------- | ----------- | ----------------------------------------- |
| **기능 적합성** | 요구사항 충족 정도  | 완전성, 정확성, 적절성                             |
| **성능 효율성** | 자원 대비 성능    | 시간 반응성(응답시간), 자원 활용성, **용량(Capacity)**    |
| **호환성**    | 타 시스템과 공존   | 공존성, **상호 운용성**                           |
| **상호작용성*** | 사용 편의성      | 학습성, 운용성, 사용자 오류 보호, **접근성**, **사용자 참여**  |
| **신뢰성**    | 기능 유지 능력    | 성숙도, 가용성, 결함 허용성(Fault Tolerance), 복구 가능성 |
| **보안성**    | 정보 보호 능력    | 기밀성, 무결성, 부인 방지, 책임 추적성, **인증성**          |
| **유지보수성**  | 수정 및 개선 용이성 | 모듈성, 재사용성, 분석성, 수정성, 시험 용이성               |
| **유연성***   | 환경 변화 적응력   | 적응성, 설치 용이성, 대체 가능성, **확장성(Scalability)** |

> *2023년 개정판에서 **Usability(사용성)**는 **Interaction Capability(상호작용성)**로, **Portability(이식성)**는 **Flexibility(유연성)**로 명칭이 변경되거나 확장되었습니다.

---

### 3. 2023년 개정판의 주요 변경 사항 (심화)

최근 기술사 시험에서는 최신 표준의 변화를 묻는 경우가 많습니다. 다음 3가지를 기억해 두세요.

1. **Safety(안전성) 추가:** 제품이 사람의 생명, 재산, 환경에 위해를 끼치지 않는 능력이 주특성 급으로 강조되기 시작했습니다.
    
2. **Scalability(확장성) 명시:** 기존에는 적응성의 일부였으나, 클라우드 환경이 보편화되면서 유연성(Flexibility)의 주요 부특성으로 중요하게 다뤄집니다.
    
3. **Resistance(저항성) 추가:** 보안성 항목에 외부 공격을 견디며 기능을 수행하는 '저항성' 개념이 보강되었습니다.
    

---
2023년 개정판에서는 기존 8대 특성에서 **안전성(Safety)**이 추가되어 총 **9대 특성**이 되었으나, 전통적인 '8대 특성' 체계 내에서의 변화를 중심으로 구성했습니다.

---

### ISO/IEC 25010 제품 품질 모델 (Product Quality Model)


> *** 주요 변경 및 참고 사항:**
> 
> - **Usability(사용성)** → **Interaction Capability(상호작용 능력)**으로 명칭 변경 (2023)
>     
> - **Portability(이식성)** → **Flexibility(유연성)**으로 명칭 변경 및 확장 (2023)
>     
> - **Safety(안전성)**가 최상위 주특성으로 신설됨.
>     

---
### ISO/IEC 25010:2023 제품 품질 모델 상세 해설

#### 1. 기능 적합성 (Functional Suitability)

시스템이 사용자의 명시적, 묵시적 요구사항을 얼마나 만족하는가에 대한 척도입니다.

- **기능 완전성 (Functional Completeness):** 사용자가 기대하는 모든 기능이 구현되어 있는가?
    
- **기능 정확성 (Functional Correctness):** 산출 결과가 정밀하고 정확한가?
    
- **기능 적절성 (Functional Appropriateness):** 사용자의 목적 달성을 위해 기능이 적절하게 제공되는가?
    

#### 2. 성능 효율성 (Performance Efficiency)

사용된 자원의 양 대비 시스템의 성능 수준을 의미합니다.

- **시간 반응성 (Time Behaviour):** 응답 시간, 처리량 등이 요구 수준을 만족하는가?
    
- **자원 활용성 (Resource Utilization):** CPU, 메모리, 네트워크 등의 자원을 효율적으로 사용하는가?
    
- **용량 (Capacity):** 시스템이 수용할 수 있는 최대 트랜잭션이나 동시 접속자 수는 충분한가?
    

#### 3. 호환성 (Compatibility)

다른 시스템과 정보나 기능을 공유할 수 있는 능력입니다.

- **공존성 (Coexistence):** 동일한 환경에서 다른 소프트웨어와 자원을 공유하며 충돌 없이 동작하는가?
    
- **상호 운용성 (Interoperability):** 타 시스템과 API 등을 통해 데이터를 주고받고 해석할 수 있는가?
    

#### 4. 상호작용 능력 (Interaction Capability) - _구 사용성(Usability)_

사용자가 시스템을 얼마나 쉽고 즐겁게 사용할 수 있는가에 초점을 맞춥니다.

- **학습성 (Learnability):** 초보 사용자가 기능을 익히기에 얼마나 쉬운가?
    
- **운용성 (Operability):** 사용자가 시스템을 제어하고 운영하기 편리한가?
    
- **접근성 (Accessibility):** 장애인, 노약자 등 모든 사용자가 차별 없이 사용할 수 있는가?
    
- **사용자 참여 (User Engagement):** 사용자의 몰입과 만족을 이끌어낼 수 있는 디자인과 경험을 제공하는가?
    

#### 5. 신뢰성 (Reliability)

의도된 시간 동안 시스템이 고장 없이 정상적으로 동작할 수 있는 능력입니다.

- **가용성 (Availability):** 사용자가 원할 때 즉시 서비스를 이용할 수 있는 상태인가? (99.9% 등)
    
- **결함 허용성 (Fault Tolerance):** 하드웨어/소프트웨어 오류 발생 시에도 운영이 유지되는가?
    
- **복구 가능성 (Recoverability):** 장애 발생 후 데이터를 복구하고 정상 상태로 돌아올 수 있는가?
    

#### 6. 보안성 (Security)

정보와 데이터를 보호하고 인가된 권한에 따라 접근을 제어하는 능력입니다.

- **기밀성 (Confidentiality):** 비인가자가 데이터에 접근할 수 없도록 보장하는가? (암호화)
    
- **무결성 (Integrity):** 데이터가 임의로 변조되지 않도록 보호되는가?
    
- **부인 방지 (Non-repudiation):** 특정 행위가 발생했음을 증명하여 사실을 부인할 수 없게 하는가? (전자서명)
    
- **저항성 (Resistance):** 외부의 공격(Hacking, DDoS 등)으로부터 스스로를 방어할 수 있는가?
    

#### 7. 유지보수성 (Maintainability)

제품의 수정, 개선, 또는 환경 변화에 따른 적응이 얼마나 쉬운가입니다.

- **모듈성 (Modularity):** 구성 요소들이 독립적으로 설계되어 한 곳의 수정이 타 곳에 영향을 덜 끼치는가?
    
- **재사용성 (Reusability):** 개발된 자산을 다른 프로젝트나 기능에서 다시 쓸 수 있는가?
    
- **시험 용이성 (Testability):** 변경된 사항을 검증하기 위한 테스트가 용이한 구조인가?
    

#### 8. 유연성 (Flexibility) - _구 이식성(Portability)_

다양한 운영 환경에 적응하거나 확장이 얼마나 용이한가입니다.

- **적응성 (Adaptability):** 다른 하드웨어나 OS 환경에 얼마나 쉽게 적응하는가?
    
- **확장성 (Scalability):** 부하 증가에 따라 시스템 규모를 유연하게 늘릴 수 있는가? (Scale-out/up)
    

#### 9. 안전성 (Safety) - _2023년 신설 주특성_

시스템 오작동으로 인해 발생할 수 있는 인명, 재산, 환경적 위험을 방지하는 능력입니다.

- **위험 식별성 (Risk Identification):** 잠재적인 위험 요소를 스스로 감지하고 식별할 수 있는가?
    
- **실패 안전성 (Fail Safe):** 장애 발생 시 시스템을 즉시 안전한 상태(Safe mode)로 전환할 수 있는가?
    


| 구분  | 주특성     | 부특성                             | 쉬운 설명                   | 생활 속 예시                  |
| --- | ------- | ------------------------------- | ----------------------- | ------------------------ |
| 1   | 기능 적합성  | 기능 완전성 (Completeness)           | 필요한 기능이 빠짐없이 들어 있는가     | 은행 앱에 조회·이체·대출 메뉴가 모두 있음 |
|     |         | 기능 정확성 (Correctness)            | 결과가 틀리지 않고 정확한가         | 계산기 앱이 소수점까지 정확          |
|     |         | 기능 적절성 (Appropriateness)        | 목적 달성에 꼭 필요한 기능인가       | 지도 앱이 최단·최적 경로를 알려줌      |
| 2   | 성능 효율성  | 시간 반응성 (Time Behaviour)         | 얼마나 빠르게 반응하는가           | 클릭 후 2초 안에 화면 로딩         |
|     |         | 자원 활용성 (Resource Utilization)   | 자원을 낭비하지 않는가            | 앱 실행 시 배터리·메모리 적게 사용     |
|     |         | 용량 (Capacity)                   | 얼마나 많이 동시에 처리 가능한가      | 수만 명 동시 접속 가능한 예매 사이트    |
| 3   | 호환성     | 공존성 (Coexistence)               | 다른 프로그램과 충돌 없이 실행되는가    | 백신과 메신저 동시 실행 가능         |
|     |         | 상호운용성 (Interoperability)        | 다른 시스템과 잘 연동되는가         | 사진 앱 → 편집 앱으로 바로 공유      |
| 4   | 상호작용 능력 | 적절성 인식성 (Recognizability)       | 이 시스템이 뭘 하는지 바로 알 수 있는가 | 첫 화면만 보고 쇼핑 앱임을 인지       |
|     |         | 학습성 (Learnability)              | 사용법을 얼마나 쉽게 배우는가        | 설명서 없이 몇 번 클릭으로 사용       |
|     |         | 운용성 (Operability)               | 조작하기 편한가                | 직관적인 메뉴 구조와 버튼 배치        |
|     |         | 사용자 오류 보호 (Error Protection)    | 실수를 막아주거나 보호하는가         | 삭제 전 “정말 삭제?” 확인창        |
|     |         | UI 미학 (Aesthetics)              | 화면이 보기 좋은가              | 가독성 좋은 글꼴과 깔끔한 색상        |
|     |         | 접근성 (Accessibility)             | 누구나 사용하기 쉬운가            | 시각장애인을 위한 음성 안내          |
|     |         | 포용성 (Inclusivity)               | 문화·언어 차별 없이 배려했는가       | 다국어 지원, 문화적 금기 배제        |
|     |         | 사용자 참여 (User Engagement)        | 계속 쓰고 싶게 만드는가           | 미션·보상으로 사용 유도            |
| 5   | 신뢰성     | 성숙도 (Maturity)                  | 오류 없이 안정적인가             | 장시간 사용해도 오류 없음           |
|     |         | 가용성 (Availability)              | 필요할 때 항상 사용 가능한가        | 24시간 서비스 중단 없음           |
|     |         | 결함 허용성 (Fault Tolerance)        | 일부 고장 나도 버티는가           | 서버 장애 시 예비 서버 자동 전환      |
|     |         | 복구 가능성 (Recoverability)         | 장애 후 복구가 가능한가           | 강제 종료 후 작업 내용 복구         |
| 6   | 보안성     | 기밀성 (Confidentiality)           | 허가된 사람만 접근 가능한가         | 의료 기록 외부 접근 차단           |
|     |         | 무결성 (Integrity)                 | 데이터가 몰래 바뀌지 않는가         | 전송 중 데이터 변조 방지           |
|     |         | 부인 방지 (Non-repudiation)         | 행위를 부인할 수 없게 하는가        | 결제 완료 기록으로 사실 증명         |
|     |         | 책임 추적성 (Accountability)         | 누가 했는지 추적 가능한가          | 로그로 수정자 확인               |
|     |         | 인증성 (Authenticity)              | 진짜 본인인지 확인하는가           | 지문·문자 인증                 |
|     |         | 저항성 (Resistance)                | 공격을 버텨내는가               | 해킹·DDoS 공격 차단            |
| 7   | 유지보수성   | 모듈성 (Modularity)                | 수정 영향이 최소화되는가           | 한 기능 수정 시 다른 기능 영향 없음    |
|     |         | 재사용성 (Reusability)              | 다른 시스템에서도 활용 가능한가       | 공통 로그인 모듈 재사용            |
|     |         | 분석성 (Analyzability)             | 원인 분석이 쉬운가              | 상세 로그로 빠른 원인 파악          |
|     |         | 수정성 (Modifiability)             | 고치고 개선하기 쉬운가            | 새 기능 추가가 쉬운 구조           |
|     |         | 시험 용이성 (Testability)            | 테스트하기 쉬운가               | 자동화 테스트로 빠른 검증           |
| 8   | 유연성     | 적응성 (Adaptability)              | 환경이 바뀌어도 잘 동작하는가        | 윈도우 앱이 맥에서도 실행           |
|     |         | 설치 용이성 (Installability)         | 설치·삭제가 쉬운가              | 클릭 몇 번으로 설치 완료           |
|     |         | 대체 가능성 (Replaceability)         | 다른 제품으로 대체 가능한가         | A 워드 문서를 B 워드에서 열기       |
|     |         | 확장성 (Scalability)               | 사용량 증가에 대응 가능한가         | 사용자 증가 시 서버 증설           |
| 9   | 안전성     | 운영 제약성 (Operational Constraint) | 위험한 사용을 제한하는가           | 로봇 작동 범위를 안전 구역으로 제한     |
|     |         | 위험 식별성 (Risk Identification)    | 위험을 미리 감지하는가            | 자율주행차가 보행자 인식            |
|     |         | 실패 안전성 (Fail Safe)              | 고장 시 안전 상태로 전이되는가       | 엘리베이터 고장 시 즉시 정지         |
|     |         | 위험 경고성 (Hazard Warning)         | 위험을 즉시 알려주는가            | 가스 누출 시 긴급 알림            |
