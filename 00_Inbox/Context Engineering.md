**컨텍스트 엔지니어링(Context Engineering)**은 AI 모델(특히 대규모 언어 모델, LLM)이 사용자의 의도를 정확히 이해하고 최적의 답변을 내놓을 수 있도록, **질문 주변의 맥락(Context)을 설계하고 관리하는 기술**을 의미합니다.

단순히 질문을 잘 던지는 '프롬프트 엔지니어링'보다 한 단계 더 나아간 개념으로, AI가 참고해야 할 데이터, 지식, 제약 조건, 그리고 대화의 흐름을 체계적으로 구조화하는 것이 핵심입니다.

---

## 1. 주요 구성 요소

컨텍스트 엔지니어링은 크게 세 가지 축으로 이루어집니다.

- **데이터 컨텍스트 (Data Context):** AI가 답변의 근거로 삼을 수 있는 외부 데이터나 지식 베이스를 제공합니다. (예: RAG 기술을 통한 문서 참조)
    
- **상태 컨텍스트 (State Context):** 이전 대화의 내용, 사용자의 선호도, 현재 진행 중인 작업의 단계 등 '기억'에 해당하 정보를 관리합니다.
    
- **지시적 컨텍스트 (Instructional Context):** AI의 역할(Persona), 답변의 형식(JSON, Markdown), 금지 사항 등을 설정하여 출력의 일관성을 유지합니다.
    

---

## 2. 왜 중요한가?

LLM은 학습된 데이터 내에서 확률적으로 답변을 생성하지만, **'지금 이 순간'** 사용자가 처한 구체적인 상황은 알지 못합니다. 컨텍스트 엔지니어링은 AI에게 "너는 지금 이런 상황이고, 이 데이터를 참고해서, 저런 형식으로 말해야 해"라는 **비즈니스 로직과 상황 판단력**을 부여합니다.

- **정확도 향상:** 환각 현상(Hallucination)을 줄이고 사실에 기반한 답변 유도.
    
- **개인화:** 사용자의 과거 이력이나 선호도를 반영한 맞춤형 응답.
    
- **일관성 유지:** 대규모 시스템에서 AI가 매번 같은 톤앤매너와 규칙을 따르도록 제어.
    

---

## 3. 프롬프트 엔지니어링과의 차이점

|**구분**|**프롬프트 엔지니어링**|**컨텍스트 엔지니어링**|
|---|---|---|
|**초점**|질문(Input)의 문구와 기법 최적화|시스템 전체의 데이터 흐름과 맥락 설계|
|**범위**|단일 메시지 또는 짧은 대화|데이터베이스(DB), API 연동, 메모리 관리 포함|
|**목표**|"어떻게 물어볼까?"|"AI에게 어떤 세상을 보여줄까?"|

---

## 4. 핵심 기술적 접근 (RAG와 연계)

최근에는 **RAG(Retrieval-Augmented Generation, 검색 증강 생성)**가 컨텍스트 엔지니어링의 핵심 수단으로 쓰입니다. 사용자의 질문에 맞는 최신 정보를 벡터 데이터베이스에서 찾아내어, 이를 프롬프트의 '컨텍스트' 영역에 주입함으로써 AI의 지식 한계를 극복합니다.

---

## 5. 효과적인 컨텍스트 설계 원칙

1. **관련성(Relevance):** 불필요한 정보는 노이즈가 됩니다. 질문과 직접적으로 관련된 정보만 선별해서 제공해야 합니다.
    
2. **구조화(Structure):** 정보를 나열하기보다 XML 태그나 JSON 형식을 활용해 AI가 각 데이터의 역할을 쉽게 인지하게 합니다.
    
3. **최신성(Recency):** 과거의 낡은 데이터보다 가장 최신의 상태 값을 우선적으로 반영합니다.
