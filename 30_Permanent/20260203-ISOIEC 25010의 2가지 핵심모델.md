
## 소프트웨어 자체의 품질 뿐만 아니라 그것을 사용하는 '경험'까지 평가한다

- 제품 품질 모델 : 시스템/소프트웨어 자체의 정적인 특성
- 사용시 품질 모델 : 실제 사용자가 특정 환경에서 사용했을 때 느끼는 품질

### 1. 제품 품질 모델
| 구분  | 주특성                                           | 부특성                             | 설명                      | 예시                       |
| --- | --------------------------------------------- | ------------------------------- | ----------------------- | ------------------------ |
| 1   | **기능 적합성(Functional Suitability)**            | 기능 완전성 (Completeness)           | 필요한 기능이 빠짐없이 들어 있는가     | 은행 앱에 조회·이체·대출 메뉴가 모두 있음 |
|     | 시스템이 사용자의 명시적, 묵시적 요굿항을 얼마나 만족하는가에 대한 척도      | 기능 정확성 (Correctness)            | 결과가 틀리지 않고 정확한가         | 계산기 앱이 소수점까지 정확          |
|     |                                               | 기능 적절성 (Appropriateness)        | 목적 달성에 꼭 필요한 기능인가       | 지도 앱이 최단·최적 경로를 알려줌      |
| 2   | **성능 효율성(Performance Efficiency)**            | 시간 반응성 (Time Behaviour)         | 얼마나 빠르게 반응하는가           | 클릭 후 2초 안에 화면 로딩         |
|     | 사용된 자원의 양 대비 시스템의 성능 수준을 의미                   | 자원 활용성 (Resource Utilization)   | 자원을 낭비하지 않는가            | 앱 실행 시 배터리·메모리 적게 사용     |
|     |                                               | 용량 (Capacity)                   | 얼마나 많이 동시에 처리 가능한가      | 수만 명 동시 접속 가능한 예매 사이트    |
| 3   | **호환성(Compatibility)**                        | 공존성 (Coexistence)               | 다른 프로그램과 충돌 없이 실행되는가    | 백신과 메신저 동시 실행 가능         |
|     | 다른 시스템과 정보나 기능을 공유할 수 있는 능력                   | 상호운용성 (Interoperability)        | 다른 시스템과 잘 연동되는가         | 사진 앱 → 편집 앱으로 바로 공유      |
| 4   | **상호작용 능력(Interaction Capability)**           | 적절성 인식성 (Recognizability)       | 이 시스템이 뭘 하는지 바로 알 수 있는가 | 첫 화면만 보고 쇼핑 앱임을 인지       |
|     | - 구 사용성(Usability)                            | 학습성 (Learnability)              | 사용법을 얼마나 쉽게 배우는가        | 설명서 없이 몇 번 클릭으로 사용       |
|     | 사용자가 시스템을 얼마나 쉽고 즐겁게 사용할 수 있는가                | 운용성 (Operability)               | 조작하기 편한가                | 직관적인 메뉴 구조와 버튼 배치        |
|     |                                               | 사용자 오류 보호 (Error Protection)    | 실수를 막아주거나 보호하는가         | 삭제 전 “정말 삭제?” 확인창        |
|     |                                               | UI 미학 (Aesthetics)              | 화면이 보기 좋은가              | 가독성 좋은 글꼴과 깔끔한 색상        |
|     |                                               | 접근성 (Accessibility)             | 누구나 사용하기 쉬운가            | 시각장애인을 위한 음성 안내          |
|     |                                               | 포용성 (Inclusivity)               | 문화·언어 차별 없이 배려했는가       | 다국어 지원, 문화적 금기 배제        |
|     |                                               | 사용자 참여 (User Engagement)        | 계속 쓰고 싶게 만드는가           | 미션·보상으로 사용 유도            |
| 5   | **신뢰성(Reliability)**                          | 성숙도 (Maturity)                  | 오류 없이 안정적인가             | 장시간 사용해도 오류 없음           |
|     | 의도된 시간 동안 시스템이 고장 없이 정상적으로 동작할 수 있는 능력        | 가용성 (Availability)              | 필요할 때 항상 사용 가능한가        | 24시간 서비스 중단 없음           |
|     |                                               | 결함 허용성 (Fault Tolerance)        | 일부 고장 나도 버티는가           | 서버 장애 시 예비 서버 자동 전환      |
|     |                                               | 복구 가능성 (Recoverability)         | 장애 후 복구가 가능한가           | 강제 종료 후 작업 내용 복구         |
| 6   | **보안성(Security)**                             | 기밀성 (Confidentiality)           | 허가된 사람만 접근 가능한가         | 의료 기록 외부 접근 차단           |
|     | 정보와 데이터를 보호하고 인가된 권한에 따라 접근을 제어하는 능력          | 무결성 (Integrity)                 | 데이터가 몰래 바뀌지 않는가         | 전송 중 데이터 변조 방지           |
|     |                                               | 부인 방지 (Non-repudiation)         | 행위를 부인할 수 없게 하는가        | 결제 완료 기록으로 사실 증명         |
|     |                                               | 책임 추적성 (Accountability)         | 누가 했는지 추적 가능한가          | 로그로 수정자 확인               |
|     |                                               | 인증성 (Authenticity)              | 진짜 본인인지 확인하는가           | 지문·문자 인증                 |
|     |                                               | 저항성 (Resistance)                | 공격을 버텨내는가               | 해킹·DDoS 공격 차단            |
| 7   | **유지보수성(Maintainability)**                    | 모듈성 (Modularity)                | 수정 영향이 최소화되는가           | 한 기능 수정 시 다른 기능 영향 없음    |
|     | 제품의 수정, 개선, 또는 환경 변하에 따른 적응이 얼마나 쉬운가          | 재사용성 (Reusability)              | 다른 시스템에서도 활용 가능한가       | 공통 로그인 모듈 재사용            |
|     |                                               | 분석성 (Analyzability)             | 원인 분석이 쉬운가              | 상세 로그로 빠른 원인 파악          |
|     |                                               | 수정성 (Modifiability)             | 고치고 개선하기 쉬운가            | 새 기능 추가가 쉬운 구조           |
|     |                                               | 시험 용이성 (Testability)            | 테스트하기 쉬운가               | 자동화 테스트로 빠른 검증           |
| 8   | **유연성(Flexibility)**                          | 적응성 (Adaptability)              | 환경이 바뀌어도 잘 동작하는가        | 윈도우 앱이 맥에서도 실행           |
|     | - 구 이식성(Portability)                          | 설치 용이성 (Installability)         | 설치·삭제가 쉬운가              | 클릭 몇 번으로 설치 완료           |
|     | 다양한 운영 한경에 적응하거나 확장이 얼마나 용이한가                 | 대체 가능성 (Replaceability)         | 다른 제품으로 대체 가능한가         | A 워드 문서를 B 워드에서 열기       |
|     |                                               | 확장성 (Scalability)               | 사용량 증가에 대응 가능한가         | 사용자 증가 시 서버 증설           |
| 9   | **안전성(Safety)**                               | 운영 제약성 (Operational Constraint) | 위험한 사용을 제한하는가           | 로봇 작동 범위를 안전 구역으로 제한     |
|     | - 2023년 신설                                    | 위험 식별성 (Risk Identification)    | 위험을 미리 감지하는가            | 자율주행차가 보행자 인식            |
|     | 시스템 오작동으로 인해 발생할 수 있는 인명, 재산, 환경적 위험을 방지하는 능력 | 실패 안전성 (Fail Safe)              | 고장 시 안전 상태로 전이되는가       | 엘리베이터 고장 시 즉시 정지         |
|     |                                               | 위험 경고성 (Hazard Warning)         | 위험을 즉시 알려주는가            | 가스 누출 시 긴급 알림            |

### 2. 사용 품질 모델 (Quality in Use)

| 구분  | 특성                            | 설명                             |
| --- | ----------------------------- | ------------------------------ |
| 1   | 유효성(Effectiveness)            | 사용자가 목표를 정확하고 완전하게 달성하는지 여부    |
| 2   | 효율성(Efficency)                | 목표 달성을 위해 소비되는 자원의 양           |
| 3   | 만족도(Satisfaction)             | 사용자의 요구와 기대가 충족되어 느끼는 만족감      |
| 4   | 위험으로부터의 자유(Freedom from Risk) | 경제적, 운영적, 환경적 위험을 최소화함         |
| 5   | 상황 포괄성(Context Coverage)      | 지정된 상황뿐만 아니라 잠재적 상황에서도 품질을 유지함 |
